本教程是基于Hexo,使用Fluid主题,托管于Github的博客教程

起步

在一切开始之前,你需要:

  • 电脑 (或Android手机)
  • Node.js
  • git
  • Hexo
  • Github账号

    1.电脑

    无论是Windows、Mac、Linux都可以制作Hexo博客。本文主要以Windows为主要说明

    2.Node.js

    Node.js 是一个基于 Chrome V8 引擎的 JavaScript Runtime

如果有兴趣,可以去官网了解
现在，我们要下载node.js,这里我链接到了Node.js的官网
下载,安装之后,打开cmd,输入 

npm -v

如果显示了版本号,则证明你安装成功了

3.Git

接下来，我们要来获取Git

Git是一个开源的分布式版本控制系统

同样,我也链接了Git的官网的下载链接,点击下载,然后正常安装即可。
再次打开cmd,输入

git --version

同样,输出了版本号便成功了。

3.Hexo

同样,官网在这——>Hexo
这次,安装的方式有些不同
首先,我们打开cmd
输入以下命令:

npm install hexo-cli -g
npm install hexo-deployer-git --save
hexo -v

如果出现了版本号,就说明安装成功了

4.Github账号

接下来,我们前往Github注册账号

一进入官网,首页便是注册,完成注册后,我们要创建仓库



到此,准备工作就完成了。

开始搭建

基础搭建

首先,在你的电脑中,选一个你喜欢的位置,创建一个文件夹,进入文件夹,右键,选择Git Bash Here

之后输入:

hexo init

接下来,Hexo会自动下载一些必要文件
下载完成后,基本的Hexo Blog系统便搭建完成了
输入hexo s来查看效果。

基本使用

  1. Hexo g 渲染网页
  2. Hexo clear清楚渲染网页
  3. Hexo d部署网页
  4. Hexo s启动本地服务器

基本配置

打开\_config.yml
这里面都是Hexo的基本设置
注意:冒号后面要加空格

基本网站信息设置

title:  #网站标题
subtitle: '' #网站副标题
description: '' #网站描述
keywords:  #网站关键字(多个关键字以','分割)
author:  #作者名称
language: #语言
timezone: '' #时区

主题设置

theme:  #输入themes文件夹的主题名

美化/优化

现在的Hexo已经基本可用
但是,原版主题并不是很美观,所以,我们要美化一下
在Hexo主题仓库中有许许多多的主题,你可以选择自己喜欢的,并根据主题官方的文档进行配置。
这里我主要以Fluid进行讲解

下载

在官网中下载,解压到\themes\文件夹中,将解压的文件名改为fluid

启用

打开\_config.yml
将theme:后内容改为fluid
再使用hexo s即可查看效果

主题配置

启动了主题之后,我们要进行主题的基本配置
详细的配置可以参照官方配置文档,这里只说一些特殊配置

Blog config.yml

highlight:
  enable: false  # 关闭默认的代码高亮

language: zh-CN  # 指定语言，可不改

主题 Config.yml

mouse_click:  # 鼠标点击动效
    enable: true
    style: love # 目前设置两个值：values（价值观词汇） | love（爱心）

color:  # 主体颜色配置，可以从这里寻找启发： https://www.webdesignrankings.com/resources/lolcolors/
  body_bg_color: "#eee"
  navbar_bg_color: "#9DC8C8"
  navbar_text_color: "white"
  text_color: "#3c4858"
  link_color: "black"
  link_hover_color: "#58C9B9"
  board_color: "#FFF"

lazyload: #懒加载,加快网页加载速度
  enable: true
  onlypost: false

index:
  banner_img: /img/default.jpg  # 首页 Banner 头图，以下相同
  banner_img_height: 100  # 头图高度，屏幕百分比，available: 0 - 100

about:  # 以下仅为页面顶部的基本信息，更多内容请在 ./pages/about.md 中编辑，支持 markdown 和 HTML
  md_path: ../../source/about.md  # 关于页文档的相对路径，可以按相对文档设置主题之外的路径，从而避免更新冲突
  banner_img: /img/default.jpg
  banner_img_height: 80  # available: 0 - 100
  avatar: /img/avatar.png # 头像
  name: 
  introduce: info / info / info # 支持 HTML
  icons: # 更多图标可从 https://fontawesome.com/v5.10.0/icons?d=gallery 查找，并以 "图标名: url" 的格式添加在下方
    "fab fa-youtube": https://space.bilibili.com/10242041
    "fab fa-github": https://github.com/Cynicink
    "fas fa-envelope":  #使用'mailto:你的邮箱'来调用邮箱发送邮件
    "fab fa-weixin":  # 微信图标，这个是特殊图标，点击不会跳转而是悬浮二维码，所以链接需要对应二维码图片地址

优化

在基本美化完成之后,我们还可以在Hexo的功能上进行一些补充

1.评论

评论功能是博客的基本功能,但由于Hexo生成的是静态网页,并不自带评论功能,这时,我们就要引入第三方的评论系统。
这里主要讲解Valine

获取APP ID 和 APP Key

首先,我们要登录或注册LeanCloud,进入控制台后点击创建应用:

应用创建后,进入应用,依次点击设置>应用Key,然后就可以看到APP ID和APP Key了:
再进入安全中心,设置安全域名。

配置

之后，来到Fluid主题的_config.yml

comments:  # 评论
    enable: true  # 开启评论
    type: valine  # 指定使用的评论模块  available: disqus | valine | gitalk | utterances | changyan | livere

valine:
    appid:  # 你的APP ID
    appkey:  # 你的APP Key
    notify: false # 邮件提醒, 详情: https://github.com/xCss/Valine/wiki
    verify: true # 验证码
    placeholder: 有什么想说的? # 评论框占位符
    avatar: identicon # gravatar style https://valine.js.org/avatar.html
    pageSize: 10 # 分页大小

这样,评论系统就完成了

2.置顶

置顶系统也是博客系统常见的功能,然而Hexo也并不自带这个功能

安装

我们使用hexo-generator-index2
使用以下命令安装:

npm install hexo-generator-index2 --save
npm uninstall hexo-generator-index --save

之后,在博客的_config.yml中加入:

index2_include_index: true 

index2_generator:
  per_page: 10 每页文章数量
  order_by:  #排序依据
    -top #此处添加top需在文章的'Front-matter'内加入top值
    -date
  exclude:
    -category psw #不包含psw分类下的文章

由于每次新建文章时都要设定top变量十分麻烦,所以我们改变一下文章模板
打开\scaffolds\post.md

---
title: {{ title }}
date: {{ date }}
tags:
categories: 
top: 1 #加入这一条,默认top为1,数字越高越靠前
---

这样,置顶功能基本完成了,但是,没有任何标识会显得不太美观,所以我们来稍微修改一下主题文件
打开\theme\fluid\layout\indes.ejs

        <p class="h4 index-header">
          <% if (post.top >= 2) { %>       //在此处插入
            <i class="fas fa-map-pin"></i>
          <% } %>
          <%= post.title %>
        </p>

之后,在置顶的文章标题前就会有一个小图钉。

部署

在本地完成了设置、美化。终于,我们可以像互联网公开博客了
这里，我们使用Github Page来托管我们的静态博客。
首先安装git插件:

npm install hexo-deployer-git --save

打开主题的_config.yml:

deploy:
  type: 'git'
  repository:
      github:  #填入之前在Github中记下的网址
  branch: master

使用hexo d进行部署
部署过程中会要求你登陆Github,按照注册Github的信息填入即可
部署完成后,登录你的Github用户名.github.io查看你的网页
到此,基本的Blog网站就搭建好了
